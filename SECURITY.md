
# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this portfolio website, please report it responsibly:

### How to Report

1. **Email**: Send details to sudiptapanda274@gmail.com
2. **Subject**: Include "SECURITY" in the subject line
3. **Details**: Provide a detailed description of the vulnerability

### What to Include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if available)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Depends on severity and complexity

### Security Measures

This portfolio implements several security best practices:

- No sensitive data exposure in frontend code
- Proper input validation and sanitization
- Secure external link handling with `rel="noopener noreferrer"`
- Regular dependency updates
- HTTPS deployment

### Scope

This security policy covers:

- The main portfolio application
- Client-side security issues
- Dependencies and third-party libraries
- Deployment security on Replit

### Out of Scope

- Third-party services and their security
- User device security
- Network-level attacks
- Social engineering attacks

## Security Updates

Security updates will be:

1. Implemented as soon as possible
2. Documented in the changelog
3. Communicated through appropriate channels

## Best Practices for Users

- Keep your browser updated
- Use secure networks when accessing the site
- Report any suspicious activity

Thank you for helping keep this project secure!
